# 测试文件结构说明

本文件夹包含项目的各类测试文件，按功能模块进行了分类整理。

## 目录结构

```
test/
├── database_tests/     # 数据库相关测试文件
├── backend_tests/      # 后端相关测试文件
├── frontend_tests/     # 前端相关测试文件（目前为空）
├── sql_scripts/        # SQL脚本文件
└── README.md           # 本文档
```

## 各文件夹内容说明

### database_tests/
包含数据库连接、表结构检查、数据操作等相关的Python测试文件。

### backend_tests/
包含后端业务逻辑测试、API测试等相关的Python测试文件。

### frontend_tests/
用于存放前端相关的测试文件（如React组件测试、Vue组件测试等）。

### sql_scripts/
包含数据库建表、索引创建、数据清理、查询测试等SQL脚本。

## 使用说明

1. 数据库测试文件通常用于验证数据库连接、表结构和数据完整性
2. 后端测试文件用于测试业务逻辑和API功能
3. SQL脚本可以通过数据库客户端直接执行
4. 所有测试文件都是非生产文件，不影响系统的正常运行

## 注意事项

- 执行测试前请确保数据库连接配置正确
- 部分测试文件可能需要特定的环境变量或配置文件
- 测试完成后可以根据需要清理测试数据
