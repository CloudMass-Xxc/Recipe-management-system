# 个性化食谱管理系统 - 启动指南

## 系统概述
个性化食谱管理系统是一个完整的Web应用，包含前端和后端两部分。前端采用React + TypeScript + Material UI构建，后端采用FastAPI构建。

## 前置条件
- 安装Node.js（推荐v16+）
- 安装Python（推荐3.9+）
- 安装PostgreSQL数据库

## 启动步骤

### 1. 准备环境

#### 克隆项目（如未克隆）
```bash
git clone [项目仓库地址]
cd LLM/Final_assignment/Vers_4
```

#### 创建虚拟环境（推荐）
```bash
# Windows
python -m venv .venv
.venv\Scripts\activate

# Linux/Mac
python3 -m venv .venv
source .venv/bin/activate
```

### 2. 启动后端服务

#### 进入后端目录
```bash
cd backend
```

#### 安装依赖
```bash
pip install -r requirements.txt
```

#### 配置环境变量
确保`.env`文件包含正确的数据库连接信息

#### 启动后端服务
```bash
# 开发模式（支持热重载）
uvicorn main:app --host 0.0.0.0 --port 8000 --reload

# 或直接运行main.py
python main.py
```

后端服务将在 http://localhost:8000 启动

### 3. 启动前端服务

#### 打开新的终端，进入前端目录
```bash
cd frontend
```

#### 安装依赖
```bash
npm install
```

#### 启动开发服务器
```bash
npm run dev
```

前端服务将在 http://localhost:5173 启动

### 4. 访问系统
- 前端应用: http://localhost:5173
- 后端API文档: http://localhost:8000/docs

## 常见问题

### 端口占用
如果端口被占用，可以修改启动命令中的端口号：
- 后端: `--port 8001`（或其他端口）
- 前端: 修改`vite.config.ts`中的端口配置

### 数据库连接错误
确保PostgreSQL服务已启动，且`.env`文件中的数据库连接信息正确。

### 依赖安装失败
- 尝试使用管理员权限运行命令
- 确保网络连接正常
- 对于Python依赖，可以尝试使用`--trusted-host`参数

## 生产环境部署

### 前端构建
```bash
cd frontend
npm run build
```
构建产物将在`dist`目录中

### 后端部署
可以使用gunicorn等WSGI服务器部署FastAPI应用

## 关闭服务
- 在终端中按 `Ctrl + C` 可以停止服务
- 或关闭终端窗口
